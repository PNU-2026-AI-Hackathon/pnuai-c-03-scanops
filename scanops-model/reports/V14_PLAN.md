작성: 2026-06-30 (V13 평가 직후, V14 설계 메모) · 지금 당장이 아니라 다음 라운드용 정리

## 0. 핵심 진단 — 왜 v13는 "정밀하지만 재현율이 낮은가"

| 벤치 | v13 재현율 | v13 오탐률 | 해석 |
|---|---|---|---|
| CVEfixes(실제 CVE) | 62.5% | 23.4% | 학습 분포 → 잘함 |
| OWASP(합성 주입) | 23.6%(LLM)/47.3%(+그래프) | 18.2% | **보수적 → 과소탐지** |

**근본 원인 = "보수적 편향(conservative bias)" 트레이드오프.**
- v13는 **CVEfixes(실제 CVE 패치)**로 학습했다. 실제 CVE는 *미묘한* 결함(누락된 경계검사·인증)이고,
  "취약/수정" 코드가 거의 똑같다. → 모델은 "확실할 때만 취약이라 외치는" 보수적 습관을 배웠다.
- OWASP는 정반대다: *명백한* 주입 패턴이지만 **서블릿 보일러플레이트 + decoy(안전인척하는 미끼)**로
  뒤덮여 있다. 게다가 우리는 **OWASP 서블릿 코드를 학습에서 100% 제외**했다(zero-shot 유지하려고).
  → v13는 "servlet식 명백한 주입" 스타일을 **한 번도 본 적이 없다** = out-of-distribution.
- 결과: 실제 CVE엔 강하지만, 합성 주입 패턴엔 과소탐지. **이건 버그가 아니라 학습 데이터 편식의 결과.**

> 사용자 관찰("그래프가 OWASP 잘 잡는 줄 알았는데 v13에서 낮아짐")의 정답:
> **그래프 기여도(+13~22건)는 그대로다. 낮아진 건 그 위에 쌓는 LLM의 baseline 재현율**
> (v12 52.7% → v13 23.6%). 즉 그래프가 나빠진 게 아니라 **LLM이 보수적이 되어** 그래프가
> 'unknown' 위임한 케이스를 LLM이 다 놓치는 것. 고칠 지점은 둘: ① LLM의 합성패턴 인지,
> ② 그래프가 더 많은 taint를 직접 'vuln' 확정(LLM 위임을 줄임).

## 1. Q2 답 — CVE/NVD 경쟁 우위(모트)는 맞다, 단 "수치로 증명"이 남았다

맞는 분석이다. 두 축의 해자(moat):
- **알려진 취약점**: CVEfixes 학습 → 흔한 CWE 클래스(주입/XSS/SSRF/역직렬화 등) 패턴 보유.
- **최신 취약점**: NVD RAG 8,883건(2026, Grok 컷오프 이후) → **Grok이 학습 못 한 신규 CVE** 인지.
- **+ 자체 호스팅**: 고객 코드 외부 유출 없음 + 호출 비용 0.

**아직 미증명**: "RAG가 신규 CVE에서 Grok을 이긴다"를 **수치로** 보여주지 못했다(현재는 정성 주장).
→ **V14 필수 실험**: `Grok 학습컷오프 이후 공개된 CVE의 코드`만 모은 held-out 벤치를 만들어
  **RAG-on vs RAG-off vs Grok** 3-way. 여기서 RAG-on이 Grok을 이기면 **모트가 정량 증명**된다.
  (지금 8,883 재임베딩이 이걸 위한 사전작업.)

## 2. Q3 답 — OWASP(합성 주입) 탐지율 올리는 법

OWASP 재현율 47%→80%+로 올리되 정밀도(FPR 18%)는 유지하는 것이 목표. 세 레버:

### 2-A. (최우선) 학습에 "합성 주입 패턴"을 다른 출처로 주입
- v13가 OWASP식 명백한 주입을 못 보는 이유는 **학습에 그런 스타일이 0건**이라서다.
- **OWASP 테스트셋은 그대로 제외**하되(zero-shot 유지), **다른 합성 주입 데이터**를 학습에 추가:
  - 후보: **CyberNative DPO**(이번 제3 벤치 출처지만 학습/평가 split 분리), **SARD/Juliet**(NIST 합성),
    또는 우리가 만든 `graph_benchmark_cases`식 taint 생성기.
  - 효과: 모델이 "request→sink 명백한 주입" 스타일을 학습 → OWASP-OOD 해소. **재현율 직접 상승.**
- ⚠️ 누수 규약: OWASP 110 코드해시 + 평가용 split은 반드시 학습 제외(v13 누수차단 로직 재사용).

### 2-B. 그래프가 더 많은 taint를 직접 'vuln' 확정 (LLM 위임 축소)
- 현재 그래프는 "확신할 때만 vuln, 애매하면 unknown→LLM". OWASP에서 unknown이 많아 보수적 LLM이 놓침.
- 개선: **다단계 taint 추적**(헬퍼 메서드·변수 경유), **full-file에서 source→sink 경로 증명** 강화.
  그래프는 고정밀이라 vuln 확정을 늘려도 FPR 거의 안 오름 → **재현율↑·정확도↑ 동시 달성.**
- 추가: 그래프 **C/C++ 커버리지** 신설(현재 미지원 → DiverseVul 등 C/C++ 벤치도 3-way 가능해짐).

### 2-C. full-file strong-safe veto 수정 (V13_BASELINE에서 발견한 버그)
- full 서블릿 파일에선 sink·sanitizer가 멀어 `strong` 플래그가 안 켜져 오탐 억제가 0이었다.
- method 추출처럼 **관련 메서드 단위로 잘라 분석**하거나, sink 주변 윈도우에서 sanitizer를 찾도록 개선.

## 3. Q1 답 — V14 로드맵: 재현율 + 정확도 동시 상승 (우선순위)

> 목표: 세 벤치(CVEfixes·OWASP·CyberNative) 모두에서 F1·재현율·정확도 **전 지표 Grok 추월.**

### 레버 A — 학습 데이터 다양화 (가장 큰 효과, §2-A와 동일)
- **멀티소스 학습셋**: CVEfixes(미묘한 실제) + 합성 주입셋(명백한 패턴) + v12 큐레이션 뱅크.
  → 미묘+명백 둘 다 학습 = 두 분포 모두 일반화. 규모 3,483 → **6,000~8,000**, CWE 커버리지 확대.
- **균형 조정**: 재현율 부족하면 vuln 비율을 50%→55~60%로 약간↑(단 FPR 감시).

### 레버 B — LoRA 용량 확대 (미묘한 판별 강화)
- 현재 target = q/k/v/o_proj만. **gate/up/down_proj(MLP) 추가** + r=32→**64**.
  → 표현력↑로 "취약 vs 수정" 미세 차이 학습 능력↑. 데이터가 커지면 과적합 위험 낮음.
- **epoch 2 체크포인트 사용**(eval_loss 최저 0.4879 < epoch3 0.4897) 또는 early-stopping.

### 레버 C — 2단계 재현율 회복(adaptive) — 정밀도 희생 없이 재현율↑
- Stage 1: v13(고정밀) 1차 판정.
- Stage 2: v13가 'safe'라 한 케이스만 **RAG 보강 + 재현율-튜닝 변형(또는 그래프 재검사)**로 2차 스캔.
  → 1단계 정밀도 유지하면서 놓친 취약을 2단계가 건짐. (v2의 adaptive 구조 재활용.)

### 레버 D — 출력 캘리브레이션
- 현재 VULN/NONE 결정적. **심각도/확신도를 출력**하게 학습 → 임계값 조절로 재현율-정밀도 곡선 확보.
- 서빙 파라미터(repeat_penalty 등)도 v13 7B는 판별력이 있으니 재현율-FPR 트레이드오프 재탐색.

### 우선순위 요약
1. **레버 A(합성 주입 데이터 추가)** — OWASP 재현율 직접 해결, 최대 효과.
2. **레버 B(LoRA 용량 + epoch2)** — 전반 판별력 강화.
3. **§1 RAG 신규-CVE 벤치** — 모트 정량 증명.
4. **레버 C(2단계) + §2-B 그래프 개선** — 정밀도 유지하며 재현율 보강.

| 벤치 | v13 LLM | v13+그래프 | Grok | 비고 |

## 4. 한 줄 결론
v13는 "정밀하지만 보수적"(실제 CVE 특화). **재현율 부족의 원인은 학습 데이터가 미묘한 실제 CVE에
편식되어 합성/명백 주입 패턴을 못 본 것.** V14는 **① 합성 주입 데이터 추가 ② LoRA 용량↑ ③ 2단계
adaptive ④ 그래프 taint 강화**로 재현율을 끌어올리면, 이미 우위인 정밀도와 합쳐 **Grok 전면 추월**이
현실적이다. 동시에 **RAG 신규-CVE 벤치로 최신성 모트를 수치 증명**한다.
