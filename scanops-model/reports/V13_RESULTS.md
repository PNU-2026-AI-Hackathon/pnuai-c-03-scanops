# ScanOps V13 결과 — 데이터 13배 확장 후 정직한 재평가

측정: 2026-06-30 · 모델 `qwen2.5-coder-security-v13-7b` (Ollama ADAPTER) · num_predict=200
> 핵심: V12(264개) 학습 부족으로 실제 CVE 판별력이 사실상 0(재현율 1.2%)이었다.
> V13는 CVEfixes 실제 코드쌍으로 학습을 **264→3,483(13배)** 확장했고, 그 결과
> **실제 CVE 판별력이 극적으로 회복(재현율 62.5%)되어 Grok을 능가**했다.

---

## 1. CVEfixes 157 (실제 2024 CVE, 10개 언어, held-out) ★핵심 벤치

| 시스템 | F1 | 재현율 | 오탐률 | 정확도 | (TP/FN/FP/TN) |
|---|---|---|---|---|---|
| v12 7B (베이스라인) | 2.5 | **1.2%** | 0.0% | 49.7% | 1/79/0/77 |
| **v13 7B** | **67.6** | **62.5%** | 23.4% | **69.4%** | 50/30/18/59 |
| Grok-3-mini | 43.7 | 32.5% | 16.9% | 57.3% | 26/54/13/64 |

- **재현율 1.2% → 62.5% (52배↑), F1 2.5 → 67.6 (27배↑), 정확도 49.7% → 69.4%.**
- **v13가 실제 CVE 벤치에서 Grok을 F1·재현율·정확도 모두에서 능가.** (Grok은 FPR만 약간 우위.)
- 그래프는 실제 CVE에 거의 'unknown'(보강 0) → 하이브리드 ≈ LLM. **순수 LLM 판별력 향상의 결과.**
- ⚠️ 정직성: held-out은 CVE-disjoint이나 cvefixes와 **同분포** → 도메인 적응 효과 포함.
  순수 일반화는 §2(OWASP zero-shot)에서 검증.

## 2. OWASP Benchmark 110 (합성 패턴, zero-shot, method 추출)

| 시스템 | F1 | 재현율 | 오탐률 | 정확도 | (TP/FN/FP/TN) |
|---|---|---|---|---|---|
| v13 7B LLM 단독 | 33.3 | 23.6% | 18.2% | 52.7% | 13/42/10/45 |
| **v13 7B + 그래프** | 57.1 | 47.3% | **18.2%** | **64.5%** | 26/29/10/45 |
| Grok-3-mini | **70.5** | 89.1% | 63.6% | 62.7% | 49/6/35/20 |

- **혼전:** Grok이 F1·재현율 우위(단, **오탐률 63.6%** — 안전 55개 중 35개 오판, "과탐").
  v13+그래프는 **정확도(64.5% > 62.7%)·오탐률(18.2% « 63.6%)** 우위 = 훨씬 정밀.
- 그래프가 LLM 놓친 취약 **13개 보강**(재현율 23.6%→47.3%).
- v13는 실제 CVE 학습으로 **보수적**이 되어 OWASP 합성 패턴엔 v12보다 덜 민감(재현율↓).
  → 실사용(실제 CVE)에 최적화된 트레이드오프.

## 2-B. CyberNative 154 (제3 벤치 — CVEfixes와 독립, 다언어, zero-shot)
> CyberNative DPO(LLM 생성 secure/insecure 쌍)에서 그래프 커버 7언어만 추출. CVEfixes와
> **출처가 완전히 달라** v13의 일반화를 정직하게 검증한다. (학습셋과 코드해시 dedup 완료.)

| 시스템 | F1 | 재현율 | 오탐률 | 정확도 | (TP/FN/FP/TN) |
|---|---|---|---|---|---|
| **v13 7B LLM 단독** | **82.6** | **92.2%** | 31.2% | **80.5%** | 71/6/24/53 |
| v13 7B + 그래프 | 77.5 | 89.6% | 41.6% | 74.0% | 69/8/32/45 |
| Grok-3-mini | 65.5 | 71.4% | 46.8% | 62.3% | 55/22/36/41 |

- **v13 LLM 단독이 모든 지표에서 Grok 완파** (F1 82.6 vs 65.5, 재현율 92.2% vs 71.4%, 정확도 80.5% vs 62.3%).
- **★일반화 증명**: CVEfixes와 무관한 출처인데도 v13 우위 → "CVEfixes 분포만 외운 것 아님" 확인.
- **재현율 92.2%** — OWASP(23.6%)와 정반대. v13 저재현율은 **OWASP 서블릿 OOD 특유 문제**지 일반 약점 아님.
- **그래프가 여기선 역효과**(F1 82.6→77.5): LLM이 이미 강할 때 그래프가 노이즈(취약 3건 오veto + 오탐 8건 추가).
  → **그래프는 만능이 아니라 상황의존**: LLM 약한 OWASP엔 도움, LLM 강한 곳엔 방해. V14에서 적응적 게이팅 필요.

## 3. 종합 해석 (정직한 결론) — 3벤치 교차

| 벤치 | 성격 | v13 최고 F1 | Grok F1 | 승자 |
|---|---|---|---|---|
| CVEfixes | 실제 CVE(미묘) | 67.6 (LLM) | 43.7 | **v13** |
| CyberNative | 독립·깨끗한 코드 | **82.6 (LLM)** | 65.5 | **v13** |
| OWASP | 합성 서블릿(decoy) | 57.1 (+그래프) | 70.5 | Grok(F1)·v13(정밀도) |

**→ 3벤치 중 2벤치에서 v13가 Grok 완파. OWASP만 예외(서블릿 OOD).**
**→ 그래프는 상황의존**: LLM 약한 OWASP만 도움, 나머진 중립~해. **현재 v13의 진짜 힘은 LLM 단독 판별력.**


| 축 | 승자 | 근거 |
|---|---|---|
| **실제 CVE(CVEfixes)** | **ScanOps v13** | F1 67.6 vs 43.7, 재현율 62.5% vs 32.5%, 정확도 69.4% vs 57.3% |
| OWASP 정확도·정밀도 | **ScanOps v13+그래프** | 정확도 64.5% vs 62.7%, FPR 18.2% vs 63.6% |
| OWASP F1·재현율 | Grok | F1 70.5 vs 57.1 (단 Grok FPR 63.6%로 과탐) |

- **핵심 주장(정직):** "자체 7B + 그래프가 **실제 CVE 대표 벤치(CVEfixes)에서 Grok을 능가**한다" — 사실.
  OWASP 합성 벤치에선 정확도·정밀도 우위, F1은 Grok(과탐 기반)이 높음.
- **V13 데이터 확장이 V12의 최대 약점(실제 CVE 판별 불가)을 해결**한 것이 가장 큰 성과.
